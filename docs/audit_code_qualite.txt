#### 🔧 Optimisations Suggérées
```php
// Remplacer lignes 16-18
$artist = $this->findArtistBySlug($slug);

// Méthode privée
private function findArtistBySlug(string $slug)
{
    $models = [Tattooer::class, Pierceur::class, Studio::class];
    
    foreach ($models as $model) {
        if ($artist = $model::where('slug', $slug)->first()) {
            return $artist;
        }
    }
    
    return null;
}
```

class EnsureClientProfile
{
    public function handle($request, $next)
    {

```php
// Service pour les stats
class TattooerStatsService
{
    public function getDashboardStats(Tattooer $tattooer): array
    {
        return BookingRequest::where('bookable_type', Tattooer::class)
            ->where('bookable_id', $tattooer->id)
            ->selectRaw('
                COUNT(CASE WHEN status = "confirmed" THEN 1 END) as completed_projects,
                COUNT(CASE WHEN status = "pending" THEN 1 END) as active_projects,
                COUNT(DISTINCT client_id) as total_clients
            ')
            ->first()
            ->toArray();
    }
}

// Policy pour autorisations
class BookingRequestPolicy
{
    public function update(User $user, BookingRequest $bookingRequest): bool
    {
        return $user->isTattooer() 
            && $bookingRequest->bookable_id === $user->tattooer->id
            && $bookingRequest->bookable_type === Tattooer::class;
    }
}
```

public function authorize(): bool
{
    return auth()->check() && auth()->user()->isClient();
}

public function rules(): array
{
    $rules = [
        'bookable_type' => 'required|string|in:' . implode(',', [
            Tattooer::class,
            Pierceur::class,
            StudioArtist::class
        ]),
    ];
    
    if ($this->bookable_type === Tattooer::class) {
        $rules['bookable_id'] = 'required|exists:tattooers,id';
    }
    // ... autres types
    
    return $rules;
}

## 4. Problèmes Transversaux

### 🔴 N+1 Queries (Critique)

**Impact**: Performance sévère sur pages chargées

**Localisations**:
- ClientController: dashboard(), messages()
- TattooerController: Toutes les méthodes de stats
- ArtistResource: hasActiveRequestForCurrentUser()

**Solution**: Eager loading systématique

### 🟡 Code Duplication (Élevé)

**Impact**: Maintenance difficile, bugs répliqués

**Exemples**:
- Vérifications autorisations dans tous les contrôleurs
- Calculs stats dupliqués
- Logiques workingHours répétées

**Solution**: Services, Policies, Traits

### 🟠 Validation Inconsistente (Moyen)

**Impact**: Sécurité et UX variables

**Problèmes**:
- authorize() parfois désactivé
- Validation XSS manquante
- Files upload validation incomplète

**Solution**: Form Requests systématiques + Policies

## 5. Recommandations Prioritaires

### 🚨 Urgent (Performance)

1. **Éliminer N+1 queries**:
   - ClientController: messages(), dashboard()
   - TattooerController: Toutes les méthodes stats
   - Resources: hasActiveRequestForCurrentUser()

2. **Service Layer**:
   - `TattooerStatsService`
   - `BookingRequestService`
   - `MessageService`

3. **Caching**:
   - Stats dashboard (cache 1h)
   - Portfolio images (cache 24h)
   - Artist listings (cache 30min)

### ⚠️ Important (Qualité)

1. **Refactoring TattooerController**:
   - Méthodes stats dupliquées
   - Autorisations via Policies
   - Validation via Form Requests

2. **Form Requests**:
   - Activer authorize() partout
   - Validation XSS systématique
   - Rules dynamiques selon type

3. **Resources API**:
   - Déplacer logique métier vers contrôleurs
   - Précharger relations
   - Éviter requêtes dans toArray()

### 📝 Améliorations (Maintenabilité)

1. **Traits réutilisables**:
   - `HasBookingOwnership`
   - `CalculatesStats`
   - `HandlesMedia`

2. **Middleware spécialisés**:
   - `EnsureClientProfile`
   - `EnsureBookingOwnership`
   - `ValidateBookingStatus`

3. **Constants centralisées**:
   - Status booking requests
   - Types de médias
   - Messages d'erreur

## Conclusion
