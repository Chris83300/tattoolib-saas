# RAPPORT D'ARCHITECTURE COMPLÈTE - TATTOOLIB SAAS

## Vue d'ensemble générale

Tattoolib SaaS est une plateforme de mise en relation entre clients et artistes (tatoueurs, perceurs, studios) avec un système complet de gestion de réservations, paiements, communications et suivi client.

## 1. Configuration Principale

### Database (config/database.php)
- **Connection par défaut**: SQLite (fallback MySQL/MariaDB/PostgreSQL/SQL Server disponibles)
- **Support Redis**: Cache et sessions
- **Foreign key constraints**: Activées par défaut
- **Transaction mode**: DEFERRED

### Authentification (config/auth.php)
- **Guard par défaut**: Web (session-based)
- **Provider**: Eloquent sur modèle User configurable
- **Sanctum**: Tokens API disponibles
- **Password reset**: Table dédiée avec expiry 60min

### Services (config/services.php)
- **Email**: Postmark/Resend/SES supportés
- **Payments**: Stripe avec webhooks
- **Notifications**: Slack intégré
- **Dépôt par défaut**: 5000 (50€) ou 30%

## 2. Architecture des Routes

### Routes Web (routes/web.php)
Structure complète avec 356 lignes organisées par rôle:

#### Routes Publiques
- `/` - Page d'accueil
- `/professionnels` - Page professionnelle
- `/marketplace` - Marketplace publique
- `/tattooer/{slug}` - Profil public tatoueur
- `/pierceur/{slug}` - Profil public perceur
- `/artistes/{slug}` - Profil public générique

#### Authentification
- `/login`, `/register` - Formulaires avec redirections par rôle
- `/register/{role}` - Inscription spécifique (client/tattooer/pierceur/studio)
- Gestion automatique des redirections selon le rôle utilisateur

#### Routes Protégées par Rôle

**Tattooer** (`/tattooer/*`):
- Profil, dashboard, demandes, calendrier
- Portfolio (upload/before-after)
- Messages, clients, paiements
- Settings (avatar/banner/schedule/password)
- Routes Livewire legacy pour compatibilité

**Client** (`/client/*`):
- Dashboard, booking-requests, chat
- Messages, réservations, paramètres

**Pierceur** (`/pierceur/*`):
- Dashboard, profil, demandes, messages
- Calendrier, paramètres, upgrade

**Studio** (`/studio/*`):
- Dashboard, profil, messages
- Calendrier, paramètres

**Studio Artist** (`/studio-artist/*`):
- Dashboard, profil, demandes
- Messages, calendrier, paramètres

#### Routes Fonctionnelles
- **Paiements acompte**: `/deposit/{bookingRequest}/*`
- **Formulaires demande**: `/booking-request/{bookableId}/{bookableType}`
- **Chat**: `/conversation/{conversation}/chat/*`
- **Auth avancée**: `/auth/*` (forgot password, etc.)

### Routes API (routes/api.php)
280 lignes d'API REST complètes:

#### Routes Publiques
- **Auth**: `/register`, `/login`
- **Tattooers**: `/tattooers/*` (index, show, portfolio, availability)
- **Piercers**: `/piercers/*` (similaire tattooers + statistics/working-hours)

#### Routes Protégées (Sanctum)

- **Messages**: CRUD avec pièces jointes et design versions

- **Piercers spécifiques**: Spécialisation, profil, horaires

#### Webhooks
- **Stripe**: `/stripe/webhook` (sans CSRF)

## 3. Modèles de Données

### User (Modèle principal)
**Rôles supportés**: client, tattooer, pierceur, studio, studio_artist, admin

**Relations polymorphiques**:
- `profile()` - Relation dynamique selon le rôle
- Relations directes: `client()`, `tattooer()`, `pierceur()`, `studio()`, `studioArtist()`

**Traits**:
- `HasApiTokens` - Sanctum
- `Billable` - Stripe Cashier
- `HasRoles` - Spatie Permissions
- `InteractsWithMedia` - Spatie MediaLibrary

**Champs principaux**:
- Authentification: email, password, timezone
- Rôle: role, status, is_active, is_admin
- Notifications: fcm_token
- Studio: studio_id, is_studio_owner, is_studio_artist
- Profil: pseudo, first_name, last_name, phone, birth_date

### Tattooer
**Relations**:
- `user()` - BelongsTo
- `studio()` - BelongsTo (optionnel)
- `workingHours()` - MorphMany
- `availabilities()` - MorphMany
- `bookingRequests()` - MorphMany
- `appointments()` - MorphMany
- `reviews()` - MorphMany

**Traits**:
- `HasSubscription` - Plans FREE/PRO
- `BookableArtist` - Réservations
- `HasCompliance` - Conformité
- `HasStripeConnect` - Paiements

**Médias**:
- `avatar` (unique)
- `banner` (unique)
- `portfolio` (multiples)
- `drawings` (multiples)
- `before_after` (multiples)

**Champs métier**:
- Professionnels: siret, siret_verified, bio, styles, address
- Stripe Connect: account_id, onboarding_complete, status
- Subscription: current_plan, is_subscribed, upgraded_to_pro_at
- Paramètres: minimum_deposit, default_deposit_rate, deadlines
- Réseaux sociaux: instagram, facebook, tiktok, website

### Client
**Relations**:
- `user()` - BelongsTo
- `bookingRequests()` - HasMany
- `appointments()` - HasMany
- `conversations()` - MorphMany
- `tattooHistory()` - HasMany
- `consent()` - HasOne

**Médias**:
- `avatar` (unique)

**Champs métier**:
- Profil: first_name, last_name, pseudo, phone, birth_date
- Historique: no_show_count, is_blacklisted, blacklist_reason
- Notes internes: notes

### BookingRequest
**Modèle central du workflow**

**Relations polymorphiques**:
- `bookable()` - MorphTo (Tattooer/Pierceur/StudioArtist)
- `client()` - BelongsTo
- `conversation()` - HasOne
- `appointment()` - HasOne
- `messages()` - HasMany

**Statuts workflow**:
- `pending` → `accepted` → `awaiting_deposit` → `deposit_paid` → `design_sent` → `confirmed`
- Alternatives: `rejected`, `expired`, `cancelled`

**Champs projet**:
- Description: tattoo_size, body_zone, description, estimated_total_price
- Préférences: preferred_timeframe, preferred_days, preferred_date, preferred_time_slot
- Prix: price_range_min/max, deposit_rate, confirmed_final_price

**Gestion avancée**:
- Design versions: included_design_versions, design_versions_used
- Modifications: modifications_per_version, modifications_used
- Deadlines: client_payment_deadline, tattooer_design_deadline
- Chat temporaire: chat_status, chat_closes_at

**Contestation**:
- dispute_status, dispute_reason, dispute_resolution
- refund_amount, dispute_opened_at, dispute_resolved_at

**Médias**:
- `reference_images` - Images de référence client

### Conversation
**Gestion cycle de vie des conversations**

**Types d'expiration**:
- `deposit_pending` - Phase acompte (7 jours par défaut)
- `permanent` - Active après paiement acompte
- `post_appointment` - Post-RDV (suppression FREE / archivage PRO)
- `archived` - Archivage permanent PRO

**Relations**:
- `messages()` - HasMany
- `lastMessage()` - BelongsTo
- `participants()` - BelongsToMany (Users avec pivot)
- `bookingRequest()` - BelongsTo

**Champs expiration**:
- expiry_type, deposit_deadline_at, expires_at
- appointment_completed_at, archived_at, is_expired
- images_preserved, expiry_warning_sent_at

### Message
**Communication structurée**

**Relations**:
- `conversation()` - BelongsTo
- `bookingRequest()` - BelongsTo
- `sender()` - BelongsTo (User)

**Types**:
- sender_type: tattooer/client
- attachment_type: image/pdf/design
- is_design_version: booléen

**Médias**:
- `attachments` - Images et PDF avec conversions (thumb, preview)

## 5. Logique Métier

### Workflow Réservation

1. **Création demande** (`pending`)
   - Client soumet formulaire avec description/préférences
   - Images de référence attachées

2. **Validation tatoueur** (`accepted`)
   - Tatoueur définit prix/dates/options
   - Modal validation avec fourchette de prix

3. **Phase acompte** (`awaiting_deposit`)
   - Chat temporaire ouvert (7 jours)
   - Deadline paiement client

4. **Paiement acompte** (`deposit_paid`)
   - Chat devient permanent
   - Deadline design tatoueur

5. **Envoi design** (`design_sent`)
   - Versions limitées (défaut: 3)
   - Modifications par version limitées

6. **Confirmation RDV** (`confirmed`)
   - Appointment créé
   - Reste à payer calculé

### Gestion Plans (FREE/PRO)

**Plan FREE**:
- 3 designs max par projet
- Chat supprimé après RDV
- Limitations fonctionnalités

**Plan PRO**:
- Designs illimités
- Chat archivé permanent
- Fonctionnalités avancées

### Expiration Conversations

**Phase deposit_pending**:
- 7 jours pour payer acompte
- Warning J-2
- Suppression automatique si expiré

**Phase permanent**:
- Active après paiement acompte
- Pas d'expiration

**Phase post_appointment**:
- FREE: Suppression immédiate
- PRO: Archivage permanent avec préservation images

### Conformité Réglementaire

- **Vérification SIRET**: Obligatoire pour tatoueurs
- **Consentements**: Forms et parental consent
- **Tracabilité**: Enregistrements et matériaux
- **Care Sheets**: Suivi client post-RDV

### Paiements Stripe

- **Stripe Connect**: Tatoueurs reçoivent directement
- **Acomptes**: 30% par défaut (configurable)
- **Webhooks**: Synchronisation automatique
- **Refunds**: Gestion contestations
